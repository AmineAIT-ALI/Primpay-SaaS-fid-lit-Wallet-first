# Objectif

Documenter l'identité visuelle de Primpay. Source de référence pour tous les supports visuels.

# Statut

⚠ Ce document est en draft. L'identité visuelle définitive est à créer après validation du nom produit.

# Principes directeurs

L'identité visuelle de Primpay doit transmettre :
- **Confiance** : le commerce local qui vous connaît.
- **Modernité sans complexité** : tech, mais accessible.
- **Proximité** : pas une grande enseigne, un outil de quartier.

# Typographie (à valider)

| Rôle | Police | Notes |
|---|---|---|
| Titres | **Inter** (Bold 700) | Propre, lisible sur mobile |
| Corps | **Inter** (Regular 400) | Cohérence full-stack |
| Monospace (codes, IDs) | **JetBrains Mono** | IDs clients, numéros |

# Logo

- Forme : à définir après validation du nom.
- Concept directeur : simple, lisible sur fond blanc ET foncé, au format 1:1 pour icône app.
- Formats requis : SVG, PNG 512×512, Favicon 32×32.

# Passes Wallet — Charte visuelle

Apple Wallet / Google Wallet ont des contraintes spécifiques :

| Élément | Contrainte | Valeur cible |
|---|---|---|
| Logo du commerçant | 160×50 px max | Logo commerçant fourni à l'onboarding |
| Couleur de fond du pass | Hex obligatoire | Couleur primaire du commerçant (ou défaut Primpay) |
| Icône strip | 320×100 px | Visuel de l'enseigne |
| Police | Gérée par iOS/Android | — |

# Ton visuel

- Pas de photos stock génériques.
- Préférer des photos réelles de commerces partenaires.
- Icônes : style outline, cohérent (ex. Lucide Icons).
- Pas de gradients complexes, pas d'ombres lourdes.

# Décisions figées

- L'identité visuelle est à finaliser après validation du nom produit.
- Le pass Wallet doit respecter strictement les guidelines Apple et Google.

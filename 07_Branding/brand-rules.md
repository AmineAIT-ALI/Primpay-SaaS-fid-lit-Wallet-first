# Objectif

Définir les règles de cohérence de marque à respecter sur tous les supports.

# Règles de langage

## Vocabulaire à utiliser

| ✅ Utiliser | ❌ Ne pas utiliser |
|---|---|
| "carte de fidélité" | "programme de fidélité" (trop corporate) |
| "Wallet" (avec majuscule) | "portefeuille digital" |
| "commerce" / "établissement" | "restaurant" (trop restrictif à la cible food) |
| "crédit" (action de tamponner) | "tampon" (trop papier) |
| "client régulier" | "end-user" |
| "gérant" / "propriétaire" | "merchant" (dans les comms publiques) |

## Ton à adopter (voir `tone.md`)

- Concret, direct, honnête.
- Jamais condescendant ("c'est simple !", "même votre grand-mère peut...").
- Pas de jargon tech dans les comms commerçants.

# Règles visuelles

- Toujours utiliser les couleurs de la palette officielle (`visual-identity.md`).
- Le logo ne peut pas être modifié, étiré, recolorisé.
- Minimum de blanc autour du logo : 20 % de sa taille.
- Jamais de logo sur fond coloré non validé.

# Règles sur les passes Wallet

- Le nom affiché dans le pass = nom exact de l'établissement (pas "Primpay").
- Le logo dans le pass = logo de l'établissement (Primpay en strip secondaire uniquement).
- Le solde de points/tampons est toujours visible en premier plan.

# Règles pour les communications terrain

- Brochures : format A5, max 3 points clés, 1 visuel fort.
- Démo : jamais de PowerPoint. Toujours sur téléphone réel.
- Email de suivi : 5 lignes max, 1 CTA, personnalisé avec le nom du commerce.

# Ce que la marque ne fait pas

- Primpay ne critique pas ses concurrents nommément dans les communications publiques.
- Primpay ne promet pas de ROI chiffré garanti.
- Primpay ne se positionne pas comme "le moins cher".

# Décisions figées

- Ces règles s'appliquent dès la validation du nom produit.
- Toute exception nécessite une validation explicite.
